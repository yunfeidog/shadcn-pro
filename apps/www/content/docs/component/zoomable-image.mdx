## Props

### ZoomableImage

| Prop           | Type      | Default | Description                                  |
| -------------- | --------- | ------- | -------------------------------------------- |
| `src`          | `string`  | `-`     | Image source URL                             |
| `alt`          | `string`  | `-`     | Alt text for the image                       |
| `width`        | `number`  | `-`     | Image width in pixels                        |
| `height`       | `number`  | `-`     | Image height in pixels                       |
| `useNextImage` | `boolean` | `true`  | Whether to use Next.js Image component       |
| `className`    | `string`  | `-`     | Additional CSS classes to apply to component |
| `zoomMargin`   | `number`  | `0`     | Margin around zoomed image in pixels         |

## Advanced Usage

### Custom Styling

```tsx showLineNumbers
<ZoomableImage
  src="/beautiful-sunset.jpg"
  alt="Sunset over mountains"
  width={500}
  height={300}
  className="border-primary/20 hover:border-primary/40 rounded-xl border-2 shadow-xl transition-colors"
  zoomMargin={40}
/>
```

### Using Native img Tag

```tsx showLineNumbers
<ZoomableImage
  src="/profile-photo.jpg"
  alt="User profile photo"
  width={200}
  height={200}
  useNextImage={false}
  className="rounded-full object-cover"
/>
```

### Gallery with Multiple Images

```tsx showLineNumbers
const images = [
  { src: "/gallery/1.jpg", alt: "Gallery image 1" },
  { src: "/gallery/2.jpg", alt: "Gallery image 2" },
  { src: "/gallery/3.jpg", alt: "Gallery image 3" },
]

function ImageGallery() {
  return (
    <div className="grid grid-cols-1 gap-4 md:grid-cols-2 lg:grid-cols-3">
      {images.map((image, index) => (
        <ZoomableImage
          key={index}
          src={image.src}
          alt={image.alt}
          width={300}
          height={200}
          className="h-48 w-full rounded-lg object-cover"
        />
      ))}
    </div>
  )
}
```

## Performance Tips

- **Next.js Optimization**: Keep `useNextImage={true}` for automatic WebP/AVIF conversion and lazy loading
- **Image Dimensions**: Always specify width and height for better layout stability
- **Responsive Images**: Use responsive classes for different screen sizes
- **Loading States**: The component handles loading states automatically when using Next.js Image
